# 学習を「続けた事実」だけを残す記録アプリ（仮）

## サービス概要
学習の習慣化が苦手な人でも無理なく続けられるよう、  
「1日1件・1項目のみ」のシンプルな学習記録に特化したWebアプリです。

プログラミング学習や資格学習など、
日々の学習内容を最小限の手間で記録し、
振り返ることで学習の継続をサポートします。

## このサービスへの思い・作りたい理由
私はプログラミング学習をはじめとした自己学習に取り組む中で、
「毎日少しは学習しているはずなのに、振り返ると何をしてきたのか分からない」
という状態に悩んでいました。

既存の学習記録サービスでは、
学習時間・教材・達成度など入力項目が多く、
学習そのものよりも「記録すること」が目的になってしまい、
次第に使わなくなってしまった経験があります。

特に自走型の学習では、
完璧な記録よりも「学習を続けている実感」を持てることの方が重要だと感じました。

既存の学習記録サービスには、
Studyplus や Notion などがありますが、
これらは学習内容・時間・教材・達成度など、
複数の情報を詳細に記録できる一方で、
入力の手間がかかり、継続のハードルが高くなりがちです。

一方、本サービスは
「1日1件・1項目のみ」の記録にあえて絞ることで、
数分かかる入力を数十秒に短縮し、
学習を続けやすい体験を提供します。

そこで本サービスでは、
**学習内容を1日1件・一言だけ記録する**
という形に絞り、
「今日はこれをやった」と最低限振り返られる、
学習特化の記録アプリを作りたいと考えました。

## ユーザー層について
- プログラミング学習や資格勉強をしている学生・社会人
- 学習記録をつけたいが、入力が面倒で続かなかった人
- RUNTEQなどの自走型学習で、日々の学習を振り返りたい人

## サービスの利用イメージ
ユーザーは1日に1回、その日に行った学習内容を記録します。  
記録内容は「今日何を学習したか」を一言で残すことを想定しています。

※同日に複数の学習を行った場合でも、
最も印象に残った学習内容を1件だけ記録することを想定しています。

過去の学習記録は一覧で確認でき、
「どんな学習をどれくらい継続できているか」を
簡単に振り返ることができます。

詳細な管理ではなく、
「とにかく学習を続けること」を目的とした利用を想定しています。  
例えば「RailsのCRUDを学習した」「SQLの復習をした」など、
学習内容を一言で記録することを想定しています。

## ユーザーの獲得について
- SNSや個人ブログでの紹介
- 「習慣化が苦手な人向け」という切り口での発信
- 実体験をベースにした利用例の共有

## サービスの差別化ポイント・推しポイント
学習記録サービスは数多く存在しますが、
多くのサービスでは以下のような特徴があります。

- 学習時間・教材・達成度など複数項目の入力が必要
- 記録の粒度が細かく、継続のハードルが高い
- 管理が目的化し、学習そのものが負担になる

本サービスでは、これらとは異なるアプローチを採用しています。

### 他サービスとの比較

| 項目 | 既存学習記録サービス | 本サービス |
| 入力項目 | 学習時間・教材・達成度など複数 | 学習内容1項目のみ |
| 記録時間 | 数分かかることが多い | 数十秒で完了 |
| 目的 | 学習管理・分析 | 学習継続の支援 |
| 想定ユーザー | 記録を細かく管理したい人 | 継続が苦手な人 |

### 本サービスの特徴
- 記録できるのは「1日1件・1項目のみ」
- 学習内容を一言で残すだけ
- 入力に迷わず、数十秒で記録が完了する

「学習を正確に管理する」ことではなく、
「学習を続けてきた事実を残す」ことに特化している点が、
他の学習記録サービスとの大きな違いです。

記録が負担になって学習が止まってしまった人にとって、
最も続けやすい学習記録体験を提供することを目指しています。

## 機能候補
### MVPリリース時
- ユーザー登録・ログイン
- 1日1件の学習記録作成
- 学習記録の一覧表示（日付順）
- 学習記録の編集・削除

### 本リリース時
- 習慣ごとの簡単な振り返り表示
- カテゴリ分け機能
- リマインダー通知
- 過去記録の検索機能

## 使用する技術スタック
- フレームワーク：Ruby on Rails
- データベース：PostgreSQL
- デプロイ：Render
